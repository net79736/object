 sellTo, buy, hold 라는 이름은 클라이언트가 객체에게 무엇을 원하는지를 명확하게 표현한다.

메시지 전송자는 메시지 수신자의 상태를 바꿔서는 안된다.
객체의 외부에서 해당 객체의 상태를 기반으로 무언가 결정을 내리는 것은 객체의 캡슐화를 위반한다.

객체지향의 기본은 함께 변경될 확률이 높은 정보와 행동을 하나의 단위로 통합하는 것이다.

 - 객체의 내부 구조를 묻는 메시지가 아니라 수신자에게 무언가를 시키는 메시지를 만들도록 프로그래밍 해라.
 - 객체들의 협력 경로를 제한하여 결합도를 낮추는 것
